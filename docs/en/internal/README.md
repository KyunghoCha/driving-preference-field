# Internal Docs Policy

This folder keeps maintainer-facing notes. It is not part of the public canonical contract.

## Purpose

Use `docs/en/internal` and `docs/ko/internal` for maintainer notes, audit results, and working vocabulary that should not silently expand the public claim surface.

## Rules

- public-facing truth belongs in explanation, reference, how-to, and status docs
- internal notes can support those docs but must not override them
- tracked audit results live under `docs/en/internal/audit` and `docs/ko/internal/audit`

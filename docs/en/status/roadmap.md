# Roadmap

This document records the current phase ordering for the repo. The active state is `Phase 5 complete, Phase 6 preparation`.

## Current position

The repo has stabilized its canonical docs, toy/runtime comparison tooling, source-adapter contract, and progression-centered score semantics enough to move into the next preparation stage.

## Phase structure

### Phase 0. Canonical docs

Fix explanation, reference, and scope documents before implementation grows.

### Phase 1. Tiny analytic evaluator skeleton

Establish a small runtime that can evaluate a field over toy inputs.

### Phase 2. Channel experiments and toy-case growth

Expand channels and cases enough to test morphology meaningfully.

### Phase 3. Local raster visualization

Render the field over the local map so morphology can be inspected.

### Phase 4. Parameter Lab

Build the comparison UI, exports, presets, and profile workflow.

### Phase 5. Source adapter

Stabilize the generic adapter contract and inspection path.

### Phase 6. Interactive studio / downstream preparation

Prepare the repo for broader experimentation without collapsing it into a planner or simulator integration layer.

## Completed

Phases 0 through 5 are complete enough to serve as the current working baseline.

## Current focus

- keep progression-centered semantics stable
- keep bilingual docs and UI help consistent with runtime behavior
- keep morphology experiments repeatable

## Next

- deeper bilingual polish
- follow-up morphology tuning where it still adds value
- downstream preparation without widening canonical scope too early

## Out of scope

Planner integration, full control loops, and production deployment remain outside the current roadmap boundary.

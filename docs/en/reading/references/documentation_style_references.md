# Documentation Style References

This page logs external documentation references that informed the repo’s writing style. The working rule still lives in [Documentation Writing Principles](../../explanation/documentation_writing_principles.md).

## 1. Diátaxis

https://diataxis.fr/

Used as the main reference for separating explanation, reference, how-to, and tutorial-style concerns.

## 2. Google Technical Writing

https://developers.google.com/tech-writing/one/paragraphs

Used for paragraph discipline, direct topic sentences, and single-question paragraph structure.

## 3. Microsoft Writing Style Guide

https://learn.microsoft.com/en-us/style-guide/welcome/

Used for clarity, concise structure, and user-facing task writing.

## 4. Kubernetes Documentation Style Guide

https://kubernetes.io/docs/contribute/style/style-guide/

Used as a strong example of task-first and reference-first separation in a large technical project.

## 5. Red Hat Modular Documentation

https://redhat-documentation.github.io/modular-docs/

Used as a reference for modular writing and stable reuse of structured sections.

## Current rule

These are supporting references only. The repo’s active writing rule is still defined by the docs in `docs/en/explanation` and `docs/ko/explanation`.
